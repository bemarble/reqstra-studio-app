# CI Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** ESLint設定ファイルの追加・GitHub Actions CI（lint + test）の構築・mainブランチ保護を設定する。

**Architecture:** `eslint.config.mjs`（Flat Config + FlatCompat）でESLintを設定し、`.github/workflows/ci.yml`でpush/PR時にlintとtestを並列実行する。mainブランチへの直接pushはgh CLIでブランチ保護ルールを設定して禁止する。

**Tech Stack:** ESLint v9 Flat Config、`@eslint/eslintrc` FlatCompat、GitHub Actions、gh CLI

## Task 1: ESLint Flat Config ファイルを作成する

**Files:**
- Create: `eslint.config.mjs`

- [ ] **Step 1: `eslint.config.mjs` を作成する**

```mjs
// eslint.config.mjs
import { FlatCompat } from '@eslint/eslintrc'
import { fileURLToPath } from 'url'
import path from 'path'

const __filename = fileURLToPath(import.meta.url)
const __dirname = path.dirname(__filename)

const compat = new FlatCompat({
  baseDirectory: __dirname
})

export default [
  {
    ignores: ['node_modules/**', 'out/**', 'dist/**', '.superpowers/**']
  },
  ...compat.extends(
    '@electron-toolkit/eslint-config-ts',
    '@electron-toolkit/eslint-config-ts/eslint-recommended',
    '@electron-toolkit/eslint-config-prettier'
  )
]
```

> **補足：** `@electron-toolkit/eslint-config-ts` はレガシー形式（CommonJS）のため FlatCompat でラップする。`@eslint/eslintrc` は ESLint v9 に同梱済みで追加インストール不要。

- [ ] **Step 2: lintが動作することを確認する**

```bash
npx eslint . 2>&1 | head -30
```

エラーがある場合は修正すべきLintエラー。`@eslint/eslintrc` が見つからない場合は次のコマンドで確認：

```bash
ls node_modules/@eslint/eslintrc/
```

存在しない場合（通常はESLint v9に同梱されているため起こらない）：

```bash
npm install --save-dev @eslint/eslintrc
```

- [ ] **Step 3: コミットする**

```bash
git add eslint.config.mjs
git commit -m "feat: ESLint v9 Flat Config を追加"
```

## Task 2: `package.json` のスクリプトを更新する

**Files:**
- Modify: `package.json`

- [ ] **Step 1: `lint` スクリプトを更新し、`lint:ci` と `test:ci` を追加する**

`package.json` の `scripts` セクションを以下のように変更する：

変更前：
```json
"lint": "eslint . --ext .js,.jsx,.cjs,.mjs,.ts,.tsx,.cts,.mts --fix",
```

変更後（`lint`, `lint:ci`, `test:ci` の3行）：
```json
"lint": "eslint . --fix",
"lint:ci": "eslint .",
"test:ci": "vitest run",
```

> **補足：** ESLint v9 Flat Config では `--ext` フラグは不要（設定ファイル内でファイルパターンを管理する）。`lint:ci` はCIで自動修正しないために `--fix` を除外。`test:ci` は vitest のウォッチモードを無効化して1回だけ実行。

- [ ] **Step 2: ローカルでスクリプトが動くことを確認する**

```bash
npm run lint:ci 2>&1 | tail -5
```

Expected: lint エラーがあれば一覧表示、なければ終了コード0で完了。

```bash
npm run test:ci 2>&1 | tail -10
```

Expected: テスト結果が表示されてプロセスが終了する（ウォッチモードにならない）。

- [ ] **Step 3: コミットする**

```bash
git add package.json
git commit -m "feat: lint:ci と test:ci スクリプトを追加"
```

## Task 3: GitHub Actions ワークフローを作成する

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: ディレクトリを作成してワークフローファイルを作成する**

```bash
mkdir -p .github/workflows
```

`.github/workflows/ci.yml` を以下の内容で作成する：

```yaml
name: CI

on:
  push:
    branches:
      - '**'
  pull_request:
    branches:
      - '**'

jobs:
  lint:
    name: lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint:ci

      - name: Type check
        run: npm run typecheck

  test:
    name: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Test
        run: npm run test:ci
```

> **補足：** `job.name` フィールドの値（`lint`, `test`）がGitHubステータスチェックの識別子になる。Task 4のブランチ保護設定でこれらの名前を参照する。`lint` と `test` は並列実行される。

- [ ] **Step 2: YAMLの構文を確認する**

```bash
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))" && echo "YAML OK"
```

Expected: `YAML OK`

- [ ] **Step 3: コミットしてpushする**

```bash
git add .github/workflows/ci.yml
git commit -m "feat: GitHub Actions CI ワークフローを追加（lint + test）"
git push origin feat/http-protocol
```

- [ ] **Step 4: GitHub Actions の実行を確認する**

```bash
gh run list --limit 3
```

Expected: `CI` ワークフローが表示される。失敗している場合はログを確認：

```bash
gh run view --log-failed
```

## Task 4: main ブランチ保護を設定する

> **前提：** GitHub リポジトリ `bemarble/reqstra-studio-app` への管理者権限が必要。gh CLI で認証済みであること。

- [ ] **Step 1: 現在のブランチ保護設定を確認する**

```bash
gh api repos/bemarble/reqstra-studio-app/branches/main/protection 2>&1 || echo "No protection set"
```

- [ ] **Step 2: ブランチ保護を設定する**

```bash
gh api repos/bemarble/reqstra-studio-app/branches/main/protection \
  --method PUT \
  --header 'Accept: application/vnd.github+json' \
  -f 'required_status_checks[strict]=true' \
  -f 'required_status_checks[contexts][]=lint' \
  -f 'required_status_checks[contexts][]=test' \
  -f 'enforce_admins=true' \
  -f 'required_pull_request_reviews=null' \
  -f 'restrictions=null'
```

Expected: 設定内容がJSONで返ってくる。エラーの場合は `gh auth status` で認証状態を確認。

> **補足：** `required_status_checks.contexts` は Task 3 で設定したjob名（`lint`, `test`）と一致させる必要がある。`enforce_admins=true` で管理者も含めてルールを適用。`required_pull_request_reviews=null` はレビュー不要の設定。

- [ ] **Step 3: 設定を確認する**

```bash
gh api repos/bemarble/reqstra-studio-app/branches/main/protection \
  | python3 -c "
import json, sys
p = json.load(sys.stdin)
print('enforce_admins:', p['enforce_admins']['enabled'])
print('required_checks:', [c['context'] for c in p['required_status_checks']['contexts']])
"
```

Expected:
```
enforce_admins: True
required_checks: ['lint', 'test']
```

- [ ] **Step 4: 直接pushが拒否されることを確認する（ドライラン）**

```bash
# main ブランチに切り替えずにローカルで確認
git push origin HEAD:main --dry-run 2>&1 | grep -E "error|reject|protected" || echo "Protection active"
```
